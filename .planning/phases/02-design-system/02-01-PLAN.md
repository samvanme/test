# Phase 2: Design System — Plan (Revised)

## Objective

Establish a **true brutalist design system** with hard shadows, bold typography, high contrast, and functional-only motion. This phase transforms the current "dark tech startup" aesthetic into an authentic brutalist visual language that communicates raw power and technical sophistication.

## Prerequisites

- Phase 1 complete (✅)
- Development environment working
- Understanding of current inline style patterns

---

## Design Philosophy

### What Brutalism Actually Is

Brutalism isn't about being ugly—it's about **honesty, function, and raw materials**. For a dark-mode AI infrastructure company:

1. **Typography as Architecture**: Massive, bold type that dominates. No glow effects—let contrast do the work.
2. **Hard Edges**: Offset shadows (not blurred), thick borders, angular forms.
3. **Color Restraint**: Near-monochrome with ONE accent color used sparingly for maximum impact.
4. **Functional Motion Only**: Animations that communicate state. Zero decoration.
5. **Visible Grid**: Structure that feels intentional and industrial.

### What We're Moving Away From

| Current (Tech Startup) | Target (Brutalist) |
|------------------------|-------------------|
| Soft cyan glows everywhere | Hard offset shadows, white borders |
| Rounded corners (rounded-full, rounded-2xl) | Sharp edges or minimal radius |
| Aurora/float decorative animations | Functional state animations only |
| Brand blue on everything | Brand blue on CTAs only |
| Outfit/Plus Jakarta Sans | Industrial neo-grotesque |
| Glass morphism cards | Solid, opaque surfaces |

### Dark Mode Brutalism Principles

Hard shadows don't work well on dark backgrounds without adaptation:
- Use **light/white borders** for elevation instead of shadows
- Use **offset shadows in brand color** for interactive elements
- Reserve **hard black shadows** for elements on lighter surfaces
- Rely on **contrast and surface color** for hierarchy

## Typography Overhaul

### Current State
```js
fontFamily: {
  sans: ['Outfit', 'Plus Jakarta Sans', 'system-ui', 'sans-serif'],
  display: ['Outfit', 'Plus Jakarta Sans', 'system-ui', 'sans-serif'],
  mono: ['JetBrains Mono', 'Fira Code', 'ui-monospace', 'monospace'],
}
```

### Target State
```js
fontFamily: {
  sans: ['Space Grotesk', 'system-ui', 'sans-serif'],
  display: ['Darker Grotesque', 'Space Grotesk', 'system-ui', 'sans-serif'],
  mono: ['IBM Plex Mono', 'ui-monospace', 'monospace'],
}
```

**Rationale:**
- **Darker Grotesque** (display): Tight, edgy neo-grotesque with brutalist energy. Variable font with 300-900 weights.
- **Space Grotesk** (body): Geometric sans with character. Not overused like Inter/Outfit.
- **IBM Plex Mono** (code): Industrial heritage, more character than JetBrains Mono.

All are available on Google Fonts.

### Task 2: Define Hard Shadow System

**Goal**: Replace soft glows with brutalist offset shadows

**File**: `src/index.css`

**Add to @layer utilities**:
```css
/* Brutalist hard shadows - offset, no blur */
.shadow-brutal-sm {
  box-shadow: 2px 2px 0 theme('colors.brand.blue');
}
.shadow-brutal {
  box-shadow: 4px 4px 0 theme('colors.brand.blue');
}
.shadow-brutal-lg {
  box-shadow: 6px 6px 0 theme('colors.brand.blue');
}

/* White border shadows for dark backgrounds */
.shadow-brutal-white-sm {
  box-shadow: 2px 2px 0 rgba(255, 255, 255, 0.9);
}
.shadow-brutal-white {
  box-shadow: 4px 4px 0 rgba(255, 255, 255, 0.9);
}

/* Hover states - shadow grows */
.hover\:shadow-brutal:hover {
  box-shadow: 6px 6px 0 theme('colors.brand.blue');
}
.hover\:shadow-brutal-lg:hover {
  box-shadow: 8px 8px 0 theme('colors.brand.blue');
}

/* Active/pressed - shadow shrinks */
.active\:shadow-brutal-sm:active {
  box-shadow: 2px 2px 0 theme('colors.brand.blue');
  transform: translate(2px, 2px);
}
```

**Verification**: Hard shadows render with proper offset
