# Phase 3: Section Redesign — Plan

## Objective

Apply **minimalist brutalist layouts** to all page sections. With design tokens established in Phase 2, this phase focuses on layout restructuring, visual hierarchy, and responsive improvements. Each section should have a distinct character while maintaining cohesive brutalist aesthetics.

## Prerequisites

- Phase 2 complete (✅)
- Design tokens available (typography, shadows, borders, spacing)
- Components already use brutalist classes

---

## Design Approach

### Layout Principles

1. **Asymmetry over symmetry** — Break out of centered layouts where impact benefits
2. **Typography as focal point** — Headlines should dominate with generous whitespace
3. **Grid intentionality** — Use 12-column grid for structure, break it for emphasis
4. **Responsive-first** — Mobile layouts should feel deliberate, not cramped
5. **Section rhythm** — Vary section heights and densities for visual flow

### What We're Changing

| Section | Current State | Target State |
|---------|---------------|--------------|
| Hero | Centered, symmetric | Left-aligned impact, asymmetric stats |
| VoiceDemo | 2-col cards | Stacked emphasis, demo prominence |
| UseCases | 3-col equal grid | Featured card + supporting grid |
| HowItWorks | 3-col steps | Horizontal timeline with visual flow |
| Stats | 4-col grid + testimonial | Data wall + prominent quote |
| FinalCTA | Centered narrow | Full-width brutalist block |
| Footer | Traditional columns | Minimal utilitarian grid |
| Header | Functional | Brutalist button styling |

### Task 10: Responsive Polish

**Goal**: Review and fix responsive issues across all breakpoints

**Files**: All component files

**Breakpoints to test**:
- 320px (small mobile)
- 375px (iPhone SE)
- 768px (tablet)
- 1024px (laptop)
- 1280px (desktop)
- 1536px (large desktop)

**Common Issues to Check**:
1. Text overflow on mobile
2. Card/grid gaps consistency
3. Padding consistency at breakpoints
4. Touch target sizes (min 44px)
5. Image scaling
6. Line length for readability (max 65ch for body text)

**Fixes might include**:
```jsx
// Text scaling
<h1 className="text-4xl sm:text-5xl md:text-6xl lg:text-7xl">

// Consistent gaps
<div className="gap-4 sm:gap-6 lg:gap-8">

// Readable line length
<p className="max-w-prose">
```

**Verification**: Site looks intentional at all breakpoints
