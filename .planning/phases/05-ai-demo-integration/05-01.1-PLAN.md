# Plan 05-01.1: Demo UI Refinements

## Objective

Polish the demo UI from Phase 5-01 to match the brutalist aesthetic and feel more like a native chat experience. Quick refinement pass focusing on visual presentation improvements.

## Execution Context

**Phase**: 5 — AI Demo Integration (sub-plan)
**Scope**: Small (4 tasks)
**Dependencies**: Plan 05-01 complete (demo architecture exists)

**Key Files to Modify:**
- `src/components/demo/ConversationTranscript.jsx` — Scrollbar styling, bottom-anchored layout
- `src/components/demo/DemoController.jsx` — Rotating tool call scenarios

**Reference:**
- `.planning/phases/05-01.1-CONTEXT.md` — Detailed requirements

## Context

### Current State
- Demo components from 05-01 are functional but visually unpolished
- Browser-default scrollbars look out of place in dark brutalist cards
- Messages fill from top-down (not chat-like)
- No avatar icons to distinguish agent vs caller
- Single simulated script per agent (no rotation)

### Target State
- Custom dark scrollbars matching brutalist aesthetic
- Bottom-anchored chat flow (new messages at bottom)
- Minimalist avatar icons with color-coded accents
- Rotating demo scenarios showcasing different tool calls

### Design Constraints
- Must maintain Revenue (blue) vs Service (white) differentiation
- Must respect existing brutalist design tokens
- Must work with reduced-motion preferences

## Tasks

### 1. Style scrollbars for dark theme
**Commit scope**: style only

Add custom scrollbar styling to ConversationTranscript:
- Thin, dark scrollbar track (matches slate background)
- Visible thumb on hover/active only
- Maintains brutalist aesthetic (no rounded corners)
- WebKit and Firefox scrollbar support

**Files**: `src/components/demo/ConversationTranscript.jsx`

---

### 2. Implement bottom-anchored chat flow
**Commit scope**: layout change

Modify ConversationTranscript for chat-like behavior:
- Use flex column-reverse for natural bottom-anchoring
- New messages appear at bottom without explicit scrolling
- Older messages scroll up and out of view
- Empty state message positioned appropriately

**Files**: `src/components/demo/ConversationTranscript.jsx`

---

### 3. Add avatar icons for participants
**Commit scope**: new UI elements

Add minimalist avatar icons to message headers:
- Bot/AI icon for agent messages (circuit/chip style)
- User/person icon for caller messages
- Color-coded by agent type (blue for revenue, white for service)
- Icons are simple, post-modern line art (not cartoon)

**Files**: `src/components/demo/ConversationTranscript.jsx`

---

### 4. Add rotating tool call examples
**Commit scope**: data and logic

Expand simulated scripts with multiple scenarios that rotate:

**Revenue Agent (Alex AI)** scenarios:
- Looking up pricing/plans
- Checking CRM for customer info
- Scheduling a demo call
- Sending a quote

**Service Agent (Sarah AI)** scenarios:
- Looking up order status
- Checking shipping/tracking
- Processing a refund
- Creating a support ticket

Rotate through scenarios on each demo replay (or random selection).

**Files**: `src/components/demo/DemoController.jsx`
